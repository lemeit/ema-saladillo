PASO 4: Verificar antes de subir
---------------------------------

Abrir cada script en scrapers/ y verificar que no haya datos sensibles
hardcodeados que no quieras hacer públicos. En este proyecto las credenciales
de Supabase están en el código — si el repositorio es PRIVADO no hay problema.
Si es PÚBLICO, considerar mover las credenciales a GitHub Secrets.

Para repositorio PRIVADO (recomendado para este proyecto):
  → Las credenciales de Supabase pueden quedarse en el código
  → El repositorio no es indexado por Google ni visible para terceros

Para repositorio PÚBLICO (opcional en el futuro):
  → Mover SUPA_URL y SUPA_KEY a GitHub Secrets
  → Referenciarlos en el código como os.environ.get("SUPA_URL")


PASO 5: Subir a GitHub
-----------------------

Desde PowerShell en la carpeta ema-saladillo\:

  git init
  git add .
  git commit -m "EMA Saladillo — commit inicial"
  git branch -M main
  git remote add origin https://github.com/TU_USUARIO/ema-saladillo.git
  git push -u origin main

Después en GitHub.com:
  → Actions → ver que el workflow "EMA Saladillo — Scrapers horarios" aparece
  → Actions → Run workflow → ejecutar manualmente para probar
  → Verificar en Supabase que llegaron datos nuevos
